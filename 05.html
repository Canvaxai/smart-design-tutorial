5. GitHubアカウントの作成とリポジトリの設定

Zero-Cost SEOの物理的な第一歩は、GitHub上にあなたの「拠点」を構築することから始まります。GitHub Pagesという強力なホスティング機能を活用することで、維持費0円の爆速サイトが手に入ります。まずは、アカウントの作成と、サイトの心臓部となる「リポジトリ（保管庫）」の設定を行いましょう。

🚀 この記事のあとの作業工程

サイトの「土地」と「建物」を確保する重要なステップです。

  1. GitHubアカウントを作成する
    → 初めての方は下部の「作成ガイド」を参照
  2. 「新しいリポジトリ」を作成する
    → プロジェクト名を決め、公開設定にする
  3. READMEファイルを追加する
    → これによりリポジトリが即座に動作します
  4. PCに作業環境（VS Code）を整える
    → 次の第6回へ進む準備をします

GitHubアカウント作成のポイント

GitHubのアカウント作成は、数分で完了します。ここでの唯一の戦略的注意点は「ユーザー名（Username）」です。デフォルトの状態では、あなたのサイトのURLが ユーザー名.github.io となるため、ブランド名に関連した、シンプルで信頼感のある名称を選びましょう。これは後に独自ドメインを設定する場合でも、管理の効率化に役立ちます。

💡 アカウント作成ガイド：

PCやスマホでの詳しい登録手順をステップバイステップで確認したい方は、以下のガイドをご覧ください。

👉 GitHubアカウント作成 完全ガイド（PC・スマホ対応）を読む

リポジトリ（Repository）を「Public」で作成する

アカウントができたら、次に「リポジトリ」を作成します。リポジトリとは、サイトを構成するすべてのファイルを保管する「箱」のようなものです。GitHub Pagesを無料で利用するためには、以下の設定を正確に行う必要があります。

  • ✅ Repository name:
    プロジェクトに関連した英数字（例：my-asset-site）を入力します。
  • ✅ Public / Private:
    必ず「Public（公開）」を選択してください。無料プランでサイト公開するには必須です。
  • ✅ Initialize this repository:
    「Add a README file」にチェックを入れます。これにより、リポジトリ作成と同時に初期ファイルが生成され、スムーズに作業を開始できます。

これで、あなたの資産サイトのための「土地」と「箱」が用意されました。次はいよいよ、手元のパソコンからこの場所を自由自在に操作するための環境を整えていきましょう。
