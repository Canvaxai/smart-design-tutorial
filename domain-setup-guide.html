独自ドメイン取得とDNS設定の完全マニュアル

GitHub Pagesで運用するサイトに独自ドメイン（.comや.jpなど）を適用する手順を解説します。この設定を行うことで、サイトの専門性とブランド力が一気に高まります。

1. ドメインを購入する

まずはドメイン管理サービスで好きな名前を購入します。国内では「お名前.com」、世界標準では「Cloudflare」がおすすめです。

  • お名前.com 公式サイト（日本語で安心）
  • Cloudflare 公式サイト（設定が高速で反映される）

2. DNS（Aレコード）を設定する

購入したサービスの管理画面（DNS設定）で、GitHubのIPアドレスを登録します。これがドメインとサーバーを結びつける作業です。

以下の4つのIPアドレスを「Aレコード」として登録してください：

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

3. GitHub側でドメインを有効化する

最後に、GitHubのリポジトリ設定（Settings > Pages）にある「Custom domain」欄に、取得したドメインを入力して保存（Save）します。反映には数分〜最大24時間かかる場合があります。

✅ ここを確認！

設定直後は「Not Found」が出ることがありますが、落ち着いて待ちましょう。反映されたら、次は第9回の「SSL化」へ進みます。
